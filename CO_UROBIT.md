## 2️⃣ Nastaviť Gitea Secrets

**Ciel:** Nastaviť prístupové tokeny, aby pipeline mohol pushovať images.

### 2.1 Vytvor Gitea Access Token
1. Prejdi na: `github.com/settings/tokens`
2. Klikni na **"Generate New Token"**
3. **Token Name:** `churchcrm-ci-cd`
4. **Permissions:**
   - [x] `read:repository`
   - [x] `write:package`
5. Klikni **"Generate Token"**
6. **Kopíruj token** (dôležité! ukáže sa len raz)

### 2.2 Vytvor Docker Hub Access Token
1. Prejdi na: [https://hub.docker.com/settings/security](https://hub.docker.com/settings/security)
2. Klikni na **"New Access Token"**
3. **Description:** `churchcrm-gitea-ci`
4. **Permissions:** `Read/Write`
5. Klikni **"Create"**
6. **Kopíruj token**

### 2.3 Pridaj Secrets do Gitea Repository
1. Prejdi do svojho repozitáru: `github.com/kolumbus120/churchcrm-docker`
2. Choď do: **Settings → Actions → Secrets → New Repository Secret**
3. Vytvor nasledujúce secrets:

| Secret Name | Value | Popis |
|-------------|-------|-------|
| `Secrets are now configured in GitHub → Settings → Secrets and variables → Actions
| `GITEA_USER` | `thasko` | Tvoje Gitea username |
| `GITEA_TOKEN` | `[token z kroku 2.1]` | Gitea access token |
| `DOCKER_HUB_USER` | `thasko` | Tvoje Docker Hub username |
| `DOCKER_HUB_TOKEN` | `[token z kroku 2.2]` | Docker Hub access token |

**✅ Overenie:**
- V Gitea Secrets by mal byť zoznam 5 secrets

## 4️⃣ Spustiť prvý build manuálne

**Ciel:** Overiť, že všetko funguje.

1. Prejdi do Gitea repozitáru
2. Choď do **Actions**
3. Vyber workflow: **"Build and Push Docker Image"**
4. Klikni **"Run workflow"**
5. Potvrď spustenie

**⏳ Čakaj 5-15 minút** (prvý build trvá dlhšie)

**✅ Overenie:**
```bash
# Skontroluj, či image existuje
docker pull github.com/kolumbus120/churchcrm-docker:latest

# Skontroluj, či je na Docker Hub
docker pull thasko/churchcrm:latest

# Zobraz images
docker images | grep churchcrm
```

**Ciel:** Skontrolovať, že pipeline bude fungovať automaticky.

1. **Skontroluj GitHub Actions logs:**

2. **Skontroluj Docker Hub:**
   - Prejdi na `https://hub.docker.com/r/thasko/churchcrm`
   - Over, že sú tam tags: `latest`, `7.3.1`, `2024-05-15` (dátum)

3. **Skontroluj image:**
   ```bash

   docker run -d -p 8080:80 --name churchcrm-test thasko/churchcrm:latest
   
   # Skontroluj, či beží

**Modern Docker image for [ChurchCRM](https://churchcrm.io) with automatic updates.**

This Docker image provides a fully functional ChurchCRM installation with:
- **PHP 8.4** + Apache with all required extensions
- **MariaDB 11.4** support (via docker-compose)
- **Automatic updates** when new ChurchCRM versions are released
- **Security patches** automatically applied from Docker Hub base images
- **Multi-architecture** support (amd64, arm64)

## 🚀 Quick Start

### Using Docker Run
```bash
docker run -d \
  --name churchcrm \
  -p 8080:80 \
  -e MYSQL_DB_HOST=mysql \
  -e MYSQL_DB_NAME=churchcrm \
  -e MYSQL_DB_USER=churchcrm \
  -e MYSQL_DB_PASSWORD=your_password \
  thasko/churchcrm:latest
```

### Using Docker Compose
See [docker-compose.yml](docker-compose.yml) for a complete setup with MariaDB.

```bash
# Clone this repository
git clone https://github.com/kolumbus120/churchcrm-docker.git
cd churchcrm-docker

# Create .env file (copy from .env.example)
cp .env.example .env

# Edit .env with your credentials
nano .env

# Start containers
docker-compose up -d
```

## 🔄 Automatic Updates

This image is **automatically updated** in the following cases:

1. **New ChurchCRM release** - Pipeline checks GitHub releases daily and rebuilds with the latest version
2. **PHP security patches** - Base image (php:8.4-apache) is automatically updated on Docker Hub
3. **OS security patches** - Debian base image receives automatic security updates

### How to Get Updates

#### Option 1: Use `:latest` tag (recommended for testing)
```yaml
services:
  churchcrm:
    image: thasko/churchcrm:latest
```

#### Option 2: Manual update (recommended for production)
```bash
# Pull new image
docker-compose pull churchcrm

# Recreate container with new image
docker-compose up -d --force-recreate
```

#### Option 3: Use Watchtower for automatic updates
```yaml
services:
  churchcrm:
    image: thasko/churchcrm:latest
  watchtower:
    image: containrrr/watchtower
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock
    environment:
      - WATCHTOWER_POLL_INTERVAL=86400 # Check every 24 hours
```

## 📞 Riešenie problémov

### Pipeline zlyhá
1. Skontroluj logs v GitHub Actions
2. Over, či sú secrets správne nastavené
3. Spusti pipeline manuálne a sleduj chybu

### Image sa nepushuje na Docker Hub
1. Over, či je Docker Hub token platný
2. Skontroluj, či má token práva Read/Write
3. Over, či je repository name správne

### Image sa nepushuje na GitHub Container Registry
1. Over, či je Gitea token platný
2. Skontroluj, či má token práva `write:package`

### Image sa nebuildne
1. Over, či je Dockerfile správne
2. Skontroluj, či nechýbajú nejaké súbory
3. Spusti `docker build .` lokálne
